## 一、项目身份与工作规则

### 项目身份

- **项目名称**：LinguaCafe-local，LinguaCafe 的本地改造版本。
- **产品方向**：sense-only FSRS 复习系统。
- **WordSense** 是实际复习对象，包含 lemma、surface_form、pos、sense_zh、sense_en、example_sentence_en、example_sentence_zh、source_chapter_id 等字段。
- **ReviewCard.target_type=sense** 是主线 — 所有复习卡管理、统计、日常复习都围绕 WordSense。
- **EncounteredWord** 只负责阅读页颜色、熟悉度总览和 legacy 兼容。
- **ReviewCard.target_type=word** 是 legacy，不应混入 sense 主线。

### 核心工作规则

1. **所有任务必须基于 GitHub 最新 master 真实代码**，不许只凭聊天记录判断代码状态。
2. **禁止使用任何 `--force` 参数**（git push --force、migrate --force 等）。
3. **禁止 `migrate:fresh`** 或清空数据库。
4. **禁止读取 `.env`**。
5. **禁止提交**：`.env`、token、数据库文件、storage 日志、`node_modules`、`vendor`、`.claude/`、临时 patch 文件（`_patch*.py`、`*.ps1`）。
6. **Laravel 测试必须顺序执行**，不要并行跑共享 testing DB 的测试套件。
7. **Vuetify 版本是 2.6.x**，禁止使用 Vuetify 3-only 的 class 或 prop。
8. **前端构建**：`npm run development`。

## 八、下一任务：C.12-c 高级筛选面板

### 任务编号

**C.12-c**

### 目标

在 `/review-cards/manage` 页面增加高级筛选面板，允许用户按 FSRS 状态、到期范围、最少复习次数、最少遗忘次数进行组合筛选。

### 新增 query 参数（GET /review-cards/manage/data）

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `fsrs_states[]` | array | `[]` | FSRS 状态多选：`new`、`learning`、`review`、`relearning` |
| `due_range` | string | `all` | 到期范围：`all`、`overdue`、`today`、`next7`、`future`、`none` |
| `reps_min` | int/null | null | 最少复习次数 |
| `lapses_min` | int/null | null | 最少遗忘次数 |

### 字段语义详解

#### fsrs_states[]
- 可选值：`new`、`learning`、`review`、`relearning`。
- 多选（数组）。
- 非法值忽略（不在白名单中的值跳过）。
- 空数组 `[]` 不过滤。
- 查询方式：`$query->whereIn('review_cards.fsrs_state', $validStates)`。

#### due_range
- `all`：不过滤。
- `overdue`：`fsrs_due_at < today start`（Carbon::today()）。
- `today`：`today start <= fsrs_due_at < tomorrow start`。
- `next7`：`now <= fsrs_due_at < now + 7 days`。
- `future`：`fsrs_due_at > now`。
- `none`：`fsrs_due_at IS NULL`。
- 非法值按 `all` 处理。
- 白名单：`['all', 'overdue', 'today', 'next7', 'future', 'none']`。

实现参考（伪代码）：
```php
$dueRange = $request->input('due_range', 'all');
$allowedRanges = ['all', 'overdue', 'today', 'next7', 'future', 'none'];
if (!in_array($dueRange, $allowedRanges, true)) {
    $dueRange = 'all';
}

switch ($dueRange) {
    case 'overdue':
        $query->where('review_cards.fsrs_due_at', '<', Carbon::today());
        break;
    case 'today':
        $query->whereBetween('review_cards.fsrs_due_at', [Carbon::today(), Carbon::tomorrow()]);
        break;
    case 'next7':
        $query->whereBetween('review_cards.fsrs_due_at', [Carbon::now(), Carbon::now()->addDays(7)]);
        break;
    case 'future':
        $query->where('review_cards.fsrs_due_at', '>', Carbon::now());
        break;
    case 'none':
        $query->whereNull('review_cards.fsrs_due_at');
        break;
    case 'all':
    default:
        break; // no filter
}
```

#### reps_min
- 非负整数。
- 非数字或空字符串不过滤。
- 查询：`$query->where('review_cards.fsrs_reps', '>=', (int) $repsMin)`。
- 使用 `ctype_digit()` 或 `is_numeric()` 校验。

#### lapses_min
- 非负整数。
- 非数字或空字符串不过滤。
- 查询：`$query->where('review_cards.fsrs_lapses', '>=', (int) $lapsesMin)`。

### 安全要求

1. `fsrs_states[]` 必须白名单校验每个值。
2. `due_range` 必须白名单校验。
3. `reps_min` / `lapses_min` 必须 `ctype_digit` 或 `is_numeric` 校验，然后 `(int)` 转换。
4. **所有高级筛选条件都在安全 query 范围内执行**（user_id/language_id/sense confirmed 已限定）。
5. 高级筛选与搜索 q、预设 filter、排序同时生效，不互相覆盖。
6. 不要用 `whereRaw` 拼接用户输入。

### 前端 UI 要求

在 `SearchAndFilter.vue` 或直接在 `ReviewCardManage.vue` 中新增：

#### 面板
- 默认折叠的高级筛选面板（可展开/收起）。
- 有筛选激活时，面板标题旁显示"已启用"chip 或 badge。

#### 控件
1. **FSRS 状态多选**：使用 Vuetify 的 `v-chip` 或 `v-btn-toggle`（multiple），选项：新卡、学习中、复习中、重新学习。对应值：`new`、`learning`、`review`、`relearning`。
2. **到期范围**：`v-select` 或 `v-btn-toggle`，选项：全部、已过期、今天、未来 7 天、未来、无到期日。对应值：`all`、`overdue`、`today`、`next7`、`future`、`none`。
3. **最少复习次数**：`v-text-field` type="number"，min="0"。
4. **最少遗忘次数**：`v-text-field` type="number"，min="0"。
5. **"应用筛选"按钮**。
6. **"清空高级筛选"按钮**。

#### 前端 Data
```javascript

    fsrsStates: [],       // 例如 ['new', 'learning']
```
